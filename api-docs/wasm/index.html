File System API

The file system API is a direct binding to the FS module provided by Emscripten and available at the package level (kuzu.FS). By default, the file system is in-memory and is not persistent, but IDBFS can be mounted optionally to provide persistent storage. The API is used to interact with the file system of the WebAssembly module. Please refer to the overview of the WebAssembly file system and the API documentation of the FS module for more information.
